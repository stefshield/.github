# Support

Thank you for using StefShield projects.
This document explains where to go for help and what to expect.

## Where to Get Help

**GitHub Discussions — for questions and general help**

Use [GitHub Discussions](https://github.com/orgs/stefshield/discussions) for:

- Questions about how to use a project
- Ideas and suggestions
- General feedback or "how do I..." questions

**GitHub Issues — for bugs and feature requests**

Use GitHub Issues for:

- Reproducible bugs
- Feature requests

Use the provided issue templates and search for existing issues before opening a new one.

## Before Opening an Issue

- Search existing issues to avoid duplicates
- Use the appropriate issue template (bug report or feature request)
- Provide enough context to allow reproduction of the issue (e.g., steps, environment, logs)

For security vulnerabilities, **do not open a public issue**. See [SECURITY.md](SECURITY.md).

## What Not to Use Issues For

Issues are reserved for bugs and feature requests.
General questions may be closed and redirected to Discussions to keep the issue tracker focused.

## Response Times

StefShield projects are maintained on a volunteer basis in one's spare time.
Response times may vary — I usually respond within a few days, but timing depends on availability.

There is no guaranteed SLA. Thanks for your patience.

## Security Issues

If you have found a security vulnerability, do not open a public issue or discussion.
Follow the process described in [SECURITY.md](SECURITY.md).

---

Thank you for your patience and for being part of the StefShield community.
